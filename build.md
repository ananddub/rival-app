STAGE 1: FOUNDATION — REWARD COIN
SYSTEM FOR RESTAURANTS

1. DEFINE YOUR COIN
• Name Ideas: GrubCoin, TastyPay, DineCash, RestoReward, MealMint, Rivl.
• Coin Type: Centralized Web2 coin, not blockchain-based yet.
• Value Peg: 1 Coin = $1 USD — simple and clear.
• Discount Rules:
- Restaurants: 15% off if paid with coins
- Grocery Stores: 2% off
• Settlement: You reimburse merchants monthly for the discount gap.
2. TECH STACK — MVP DEVELOPMENT
Customer Mobile App (Flutter or React Native):
- Coin Wallet (Balance, History)
- QR Payment / Scan-to-Pay
- Buy Coins (via Stripe/UPI)
- Restaurant & Offer Listings
Merchant Dashboard:
- Accept Payments, Track Transactions
- Monthly Reports, Offer Configuration
Admin Dashboard:
- Monitor Coins, Manage Restaurants, Users, Settlements, and Promotions
Tech Stack:
- Backend: Firebase / Supabase
- Frontend: Flutter or React Native
- Payments: Stripe / Razorpay / Plaid
- Auth: Firebase Auth / Supabase Auth
3. ONBOARDING RESTAURANTS
What You Offer Restaurants:
- Free Loyalty System, Increased Loyalty, Free Listing
Terms with Restaurants:
- Customers pay in Coins → Restaurant gives % discount
- You settle the discounted amount monthly
- Simple onboarding via dashboard or sales team
■ HOW YOU MAKE MONEY
Revenue Stream Description
1. Coin Sales Users buy coins with real money. You hold that cash until coins are redeemed.
2. Breakage Profit Some coins are never used — this is pure profit.
3. Future Add-ons Ads, promoted listings, premium features, analytics.
